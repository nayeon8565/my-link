# 프로젝트 요구사항 정의서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 사용자가 여러 개의 링크를 하나의 페이지에 모아서 공유할 수 있는 '멀티 링크 프로필' 서비스 구축.
- **대상 사용자:** 
  - 본인의 작업물, 포트폴리오, 창작물, 운영 채널 등을 종합적으로 공유하고 싶은 개발자 및 크리에이터.
- **디자인/기술 환경:** `shadcn/ui` 기반 컴포넌트, `Firebase` (인증 및 데이터베이스) 활용, **테마(다크/커스텀) 지원 없음(단일 스타일)**.

## 2. 핵심 기능 목록 (MVP)

### 사용자 인증 및 기본 프로필 설정
- **구글 간편 기반 자동화:** 복잡한 정보 입력 없이 Firebase Auth 기반 **Google 소셜 단독 로그인 지원**. (회원가입/로그인 통합 프로세스)
- **이미지 자동 연동:** 사용자 프로필 이미지는 별도 설정창 없이 구글 소셜 계정의 기본 프로필 사진을 연동하여 화면에 표시.
- **고유 URL 슬러그 방식 자동화 (`displayName`):**
  - 접속 고유 URL 슬러그(`mylink.com/displayName`)로 사용되는 **`displayName(디스플레이 네임)`** 은, 최초 가입 시 구글 지메일 아이디의 앞부분(이메일 `@` 기준 앞부분)을 추출하여 **자동으로 초기값 설정**. (예: `abcd123@gmail.com` -> `abcd123`)
  - 이후 대시보드 화면에서 사용자가 직접 인라인 편집으로 수정 가능.
- **화면 노출용 `username(유저네임)`:** 방문자 화면 중앙에 크게 보이는 실제 이름 혹은 활동명.
- **인라인(In-line) 수정 액션:** 프로필 소개글(`bio`), `username`, `displayName` 등의 모든 텍스트 정보를 화면에서 그대로 클릭 후 수정하는 **인라인 편집 UX** 적용.

### 링크 목록 관리
- **링크 엔티티 관리:** 외부 연결 대상 URL과 노출용 타이틀 텍스트 추가/삭제 기능.
- **모든 항목 인라인 수정:** 등록된 링크 타이틀이나 URL도 클릭하여 즉각 반영되는 인라인 편집 지원.
- **파비콘(Favicon) 구글 API 노출 연동:** 링크가 추가/수정될 시 URL 주소를 파싱해 **구글 퍼블릭 파비콘 API(`https://s2.googleusercontent.com/s2/favicons?domain=...`)** 의 이미지 소스를 즉시 불러와 링크 블록 옆 아이콘으로 노출.
- **링크 순서 제어:** 드래그 앤 드롭을 통한 리스트 위아래 순서 변경 및 특정 사이트 일시 숨김(비활성화) 토글 스위치.

### 뷰어 페이지 및 방문자 통계
- **통계 대시보드:** 내가 생성한 프로필 페이지로의 전체 방문 트래픽 수, 그리고 각 링크별 유도된 누적 클릭 카운트 정보를 통합하여 조회 가능.

## 3. 기능 및 데이터베이스 파이프라인

### 3.1. 자동 가입 초기 렌더링
- 신규 접속자가 구글 로그인을 수행하면, 구글 반환 계정 정보 중 이메일의 로컬파트(예: `dev.tester@gmail.com` -> `dev.tester`)를 식별하여 **`displayName(URL 슬러그)` 초기값**으로 지정해 DB에 적재합니다.
- 만약 해당 지메일 앞부분이 이미 타 사용자의 `displayName`으로 선점되어 중복될 경우, 난수나 임의의 식별자를 덧붙여 고유성을 확보하는 중복방지 프로세싱 로직이 백그라운드에 포함됩니다.

### 3.2. Firestore 구조 설계 (서브 컬렉션 지향)
- **`users` (사용자 전역 컬렉션):** Firebase 구글 소셜 인증 내역을 기반으로 자동 반환된 `photoURL`, 구글 계정명 파생 `displayName(URL용 고유 슬러그)`, `username(화면 표시용 이름)`, `bio(소개글)` 필드들을 최상단 사용자 문서 안에 보관합니다.
- **`links` (링크 서브컬렉션 분리 모델):** 사용자가 저장하는 모든 항목 단위의 링크 문서들은 개별 사용자의 `users/{userId}/links` 하위 경로를 가지는 **서브 컬렉션** 체계를 구성합니다. 이를 통해 링크들의 자체적인 속성 변경과 빠른 조회, 그리고 통계 카운트 확장을 유연하게 보장합니다.

### 3.3. 관리자 플랫폼 UX 철학
- **통합 인라인 편집 모델:** 별도의 모달(Modal) 창이나 설정 팝업을 거치지 않게 하여 사용자 동선을 아낍니다. 텍스트(이름/안내글/서브URL 등)를 클릭해 수정하고, 포커스가 해제(blur)되는 즉시 Firestore와 통신하여 데이터를 덮어씌우는 Seamless UI 철학입니다.

### 3.4. 외부 공개용 웹 프로필 뷰어
- **테마제한(Themeless) 배포 디자인:** 화려하고 복잡한 커스텀을 지양하고, 오직 하나의 통일된 명도/채도 디자인 레이아웃 컨센서스(예: shadcn/ui 기반 모던/클린 스타일) 패턴으로 배포됩니다.
- 공유 최적화(SEO) 대응을 위해 서버사이드나 Meta Data 설정을 통해 Open Graph (OG) 태그 정보를 세팅하여, 링크 메신저 전달 시 `username`과 `bio` 구문이 카드 썸네일 형태로 자동 반영되게 지원합니다.
